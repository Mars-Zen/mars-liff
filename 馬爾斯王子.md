# 馬爾斯王子 — 專案概覽與遷移指南

## 這是什麼專案？

**馬爾斯王子（星語守護者）** 是一個以 LINE 為載體的占星與靈性服務聊天機器人平台。使用者完全透過 LINE（台灣 / 亞洲地區主流通訊軟體）進行互動。整個系統 100% 建立在 **n8n** 上，沒有傳統意義上的應用程式原始碼。

---

## 運作方式（架構）

```
使用者傳送 LINE 訊息
        ↓
LINE Messaging API → n8n Webhook（入口工作流）
        ↓
n8n 依訊息類型路由至對應工作流
        ↓
工作流呼叫外部 API（Gemini AI、天氣等）
        ↓
n8n 讀寫內建資料表
        ↓
n8n 透過 LINE Reply API 回覆使用者
```

前端（LIFF 頁面）為靜態 HTML 檔案，托管於 GitHub Pages，透過 LIFF（LINE 前端框架）在 LINE 內開啟，直接以 HTTP POST 呼叫 n8n Webhook。

---

## 技術架構

| 層級 | 技術 |
|------|------|
| 自動化引擎 | n8n（自架） |
| 通訊管道 | LINE Messaging API |
| 前端介面 | 靜態 HTML/CSS/JS，透過 LINE LIFF 開啟，托管於 GitHub Pages |
| AI 模型 | Google Gemini 2.5 Flash |
| 資料庫 | n8n 內建資料表 |
| 檔案儲存 | Google Cloud Storage（精靈／角色圖片） |
| 內部 AI 模型 | 本地 LLM 服務，位於 `host.docker.internal:3000`（必須與 n8n 在同一台 VM 上執行） |
| 占星計算 | 內部 Python API（`PY_ASTRO_API_KEY`） |

## 所需憑證與變數

### n8n Credentials（由 MIS / 管理員統一管理）
| 憑證 | 使用於 |
|------|--------|
| LINE Channel Access Token | 所有 LINE 回覆 / 推播工作流 |
| Google Gemini API | Guardian-Agent、Natal、OnDemand Fortune 等 |
| OpenWeatherMap API | OnDemand Daily Outfit |
| Google Cloud Storage OAuth2 | Figure 工作流（圖片儲存） |

| 變數 | 使用於 |
|------|--------|
| `GEMINI_API_KEY` | Create Pet（直接 HTTP 呼叫 Gemini） |
| `PY_ASTRO_API_KEY` | Natal、OnDemand Solar Return、OnDemand Fortune 等 |

### 內部服務（必須與 n8n 在同一台 VM 上執行）

## 遷移步驟摘要

### 基礎設施人員負責：
1. 在 GCP 上開一台 VM
2. 安裝 Docker，用 Docker Compose 跑 n8n
3. 設定網域和 HTTPS（LINE 不接受純 IP 或 http）
4. 提供：新 n8n 網址 + 管理員登入帳號密碼

### 開發者負責（基礎設施就緒後）：
1. 從現有 n8n Cloud 匯出所有 45 個工作流 JSON
2. 匯入新 n8n 環境
3. 重建所有資料表，並從 CSV 備份匯入資料
4. 更新工作流中的資料表 ID（遷移後 ID 會改變）
5. 設定 Credentials 與 Variables（或交由 MIS 處理）
6. 在 LINE Developers Console 更新 Webhook URL 為新 n8n 網址
7. 更新所有 HTML 檔中的 API 網址（舊 → 新 n8n 網域）
8. 測試所有主要功能流程

---

## 重要注意事項

- LINE 機器人 Channel 目前是在個人 LINE 帳號下建立的，正式上線前應將擁有權轉移至公司帳號
- 本地 LLM 服務（`host.docker.internal:3000`）必須與 n8n 在**同一台 VM** 上執行，否則需更新相關工作流的網址設定
- 遷移期間，舊 n8n Cloud 環境（`webbermars.app.n8n.cloud`）可同時保持運作，確保安全過渡
